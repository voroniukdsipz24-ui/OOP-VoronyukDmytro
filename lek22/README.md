### JSON vs XML: практичне порівняння

У сучасному програмуванні дуже важливу роль відіграє серіалізація даних — процес перетворення об’єктів у формат, який можна зберігати у файлі або передавати по мережі. Найпоширенішими форматами серіалізації є JSON та XML. Обидва формати підтримуються у .NET та часто використовуються при роботі з файловим I/O, веб-сервісами та конфігураційними файлами.

JSON (JavaScript Object Notation) — це легкий текстовий формат обміну даними, який широко використовується у веб-розробці та REST API. XML (eXtensible Markup Language) — більш старий, але дуже гнучкий формат, який часто використовується у корпоративних системах, SOAP-сервісах та складних конфігураціях.

---

### Приклад класу для серіалізації

```csharp
public class Student
{
    public int Id { get; set; }
    public string Name { get; set; }
    public int Age { get; set; }
    public string Group { get; set; }
    public double AverageMark { get; set; }
}
```

---

### Серіалізація в JSON

```csharp
using System.Text.Json;

Student st = new Student
{
    Id = 1,
    Name = "Ivan",
    Age = 20,
    Group = "CS-21",
    AverageMark = 4.5
};

var options = new JsonSerializerOptions
{
    WriteIndented = true
};

string json = JsonSerializer.Serialize(st, options);

File.WriteAllText("student.json", json);
```

Результат JSON:

```json
{
  "Id": 1,
  "Name": "Ivan",
  "Age": 20,
  "Group": "CS-21",
  "AverageMark": 4.5
}
```

---

### Серіалізація в XML

```csharp
using System.Xml.Serialization;

XmlSerializer serializer =
    new XmlSerializer(typeof(Student));

using (StreamWriter writer =
       new StreamWriter("student.xml"))
{
    serializer.Serialize(writer, st);
}
```

Результат XML:

```xml
<Student>
  <Id>1</Id>
  <Name>Ivan</Name>
  <Age>20</Age>
  <Group>CS-21</Group>
  <AverageMark>4.5</AverageMark>
</Student>
```

---

### Порівняння JSON та XML

| Критерій      | JSON        | XML              |
| ------------- | ----------- | ---------------- |
| Розмір        | Менший      | Більший          |
| Читабельність | Висока      | Середня          |
| Швидкість     | Вища        | Нижча            |
| Валідація     | JSON Schema | XSD              |
| Коментарі     | Немає       | Є                |
| Простори імен | Немає       | Є                |
| Популярність  | Веб, REST   | Enterprise, SOAP |

JSON має простіший синтаксис і займає менше місця, тому він швидше обробляється програмами. XML більш багатослівний, але має розширені можливості, такі як простори імен та перевірка схеми.

---

### Реальні сценарії використання

1. Використання JSON
   JSON найкраще підходить для веб-додатків, REST API та мобільних програм.
   Наприклад, більшість сучасних сайтів отримують дані від сервера саме у форматі JSON.

2. Використання XML
   XML часто використовується у банківських системах, державних сервісах та великих enterprise-проєктах, де важлива строгість структури та перевірка даних.

3. Конфігураційні файли
   XML часто використовується у старих системах, а JSON — у сучасних застосунках.

---

### Чому JSON став домінуючим

JSON став основним форматом для веб-додатків через:

* простий синтаксис
* менший розмір файлів
* швидку обробку
* зручну інтеграцію з JavaScript
* хорошу підтримку у REST API

Оскільки більшість сучасних сервісів працюють через HTTP API, JSON став стандартом.

---

### Чому XML досі використовується

XML залишається важливим у enterprise-середовищі через:

* підтримку XSD-схем
* підтримку namespace
* можливість складної структури
* сумісність зі старими системами
* використання у SOAP-сервісах

У великих корпоративних системах важлива строгість структури, тому XML досі актуальний.

### Контрольні питання

1. У чому різниця між синхронним та асинхронним читанням файлів?
   Синхронне читання виконується послідовно і блокує виконання програми до завершення операції. Асинхронне читання дозволяє програмі продовжувати роботу, поки файл читається у фоновому режимі. Асинхронний підхід рекомендується використовувати у UI-додатках та при роботі з великими файлами.

2. Які переваги має JSON над XML і навпаки?
   JSON має менший розмір, простіший синтаксис і швидше обробляється, тому частіше використовується у веб-додатках. XML підтримує схеми валідації, простори імен та складні структури, тому використовується у великих корпоративних системах.

3. Як ігнорувати властивість при серіалізації в JSON та XML?
   У JSON використовується атрибут `[JsonIgnore]`, а в XML — `[XmlIgnore]`.
   Ці атрибути дозволяють не включати певні властивості класу у серіалізований файл.

4. Чому важливо використовувати using при роботі з потоками?
   Конструкція `using` гарантує правильне закриття файлу після завершення роботи. Це запобігає витоку ресурсів і помилкам доступу до файлу.

5. Як обробляти помилки при десеріалізації некоректного JSON?
   Потрібно використовувати блок `try-catch` та перехоплювати виключення типу `JsonException`, `IOException` або `FileNotFoundException`. Це дозволяє програмі не завершитися аварійно і правильно повідомити про помилку.

---

### Висновок

JSON і XML — це два основних формати серіалізації даних, які використовуються у .NET та інших мовах програмування. JSON є простішим, швидшим і зручнішим для сучасних веб-додатків, тому він став домінуючим форматом. XML є більш складним, але має розширені можливості, тому він залишається важливим у корпоративних системах та складних інтеграціях. Правильний вибір формату залежить від задачі, вимог до структури даних та середовища, у якому працює програма.
